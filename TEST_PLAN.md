# 副業支援エージェント - テスト計画

## 📋 概要

このドキュメントでは、副業支援エージェントの包括的なテスト戦略を定義します。

---

## 🎯 テスト目標

1. **機能テスト**: 各ワークフローが意図通りに動作するか
2. **パフォーマンステスト**: 応答時間が許容範囲か
3. **ユーザビリティテスト**: ユーザーが簡単に使えるか
4. **信頼性テスト**: 提案の品質が一定水準か（回答の妥当性）
5. **エラーハンドリングテスト**: 例外発生時の対応が適切か

---

## 📝 テストケース

### 1. ユーザープロファイリング テスト

#### TC-01: 基本的なスキル入力

```
前提条件:
  - エージェントが起動している
  - ユーザーが初回ユーザー

操作手順:
  1. 「初回カウンセリング開始」をクリック
  2. スキルフィールドに「Python」を入力
  3. 経歴フィールドに「3年のソフトウェア開発」を入力
  4. 利用可能時間に「週10時間」を入力
  5. 目標月収に「50,000円」を入力
  6. 「次へ」をクリック

期待結果:
  ✅ 入力が正常に受け取られる
  ✅ バリデーションエラーなし
  ✅ ステップ 2 に進む
  ✅ 入力値が次ステップで引き継がれている

失敗条件:
  ❌ エラーメッセージが表示される
  ❌ 入力値が消える
  ❌ 先に進めない
```

#### TC-02: スキル複数入力

```
操作手順:
  1. スキルフィールドに複数入力: "Python", "SQL", "Data Analysis"
  2. 「次へ」をクリック

期待結果:
  ✅ 複数スキルが配列として保存される
  ✅ 後のステップで複数スキルが考慮されている
     例: "あなたの Python + SQL のスキルは〜"

テスト項目:
  - 最大入力数: 10個まで
  - 重複チェック: 同じスキルが2回入力された場合
  - スキル名の正規化: "python" vs "Python" の判定
```

#### TC-03: 必須項目チェック

```
操作手順:
  1. 必須項目の1つ（例：スキル）を空白のままに
  2. 「次へ」をクリック

期待結果:
  ✅ エラーメッセージが表示される
  ✅ 「スキルを入力してください」と明記
  ✅ 該当フィールドが赤くハイライト
  ✅ その他のフィールドの入力値は保持される

テスト項目:
  - スキル（必須）
  - 経歴（必須）
  - 利用可能時間（必須）
  - 目標月収（必須）
```

### 3. Web検索 テスト

#### TC-07: キーワード生成テスト

```
入力例:
  スキル: "Python, Data Analysis"
  業界: "FinTech"

操作手順:
  1. プロファイリング完了後、Web検索が開始
  2. 生成されたキーワードをログで確認

期待結果:
  ✅ 自動生成キーワード例:
     - "Python Data Analysis 副業 単価 2026"
     - "Python freelance rate USD JPY"
     - "FinTech 市場動向"
     - "Upwork Python Data Analysis"
  ✅ キーワード数: 4～6個

テスト項目:
  - キーワード多様性: 異なるテーマをカバーしているか
  - 言語混在: 日本語 + 英語の最適な配分
  - 具体性: 一般的すぎないか
```

#### TC-08: Web検索実行テスト

```
前提条件:
  - インターネット接続が利用可能

操作手順:
  1. キーワード「Python 副業 単価 2026」で検索実行
  2. 結果の件数を確認

期待結果:
  ✅ 検索成功
  ✅ 結果件数: > 100件
  ✅ 信頼度スコアが自動算出される
  ✅ 上位5件が WebFetch 対象として選定

エラーシナリオ:
  - タイムアウト (30秒以上かかる)
    → キャッシュから返す + 警告表示
  - 0件検出
    → 類似キーワードで自動リトライ
  - 403エラー (ロボット検出)
    → リトライ待機 (60秒) + キャッシュ使用

テスト項目:
  - 並列実行数: 最大3つまで
  - タイムアウト時間: 30秒
  - リトライ回数: 最大3回
```

#### TC-09: WebFetch - Upwork データ抽出

```
前提条件:
  - Upwork にアクセス可能

操作手順:
  1. Web検索で「Upwork」関連の結果を取得
  2. WebFetch で Upwork ページを取得
  3. 抽出データを確認

期待結果:
  ✅ 取得成功
  ✅ 以下データが抽出される:
     - 案件数
     - 報酬範囲 ($25/h ～ $150/h)
     - スキル要件
     - 競争度（low/medium/high）

テスト項目:
  - HTML パース精度: ≥ 90%
  - 構造化データ品質: キーフィールド全て取得
  - エラーハンドリング: ページ構造変更への対応

テストケース:
  A) スキル: Python
     期待: データ件数 > 1000
  
  B) スキル: 非常にニッチな分野
     期待: 0件 → 代替提案を表示
```

#### TC-10: キャッシュと最新性のバランス

```
操作手順:
  1. 月初にユーザーAがカウンセリング実行
  2. Web検索結果がキャッシュに保存される
  3. 月半ばにユーザーAが再度実行
  4. キャッシュ TTL (7日) 内か確認

期待結果:
  ✅ TTL内: キャッシュから返す
  ✅ TTL超過: 新規検索実行
  ✅ ユーザーに「XXX 前の情報」と表示

テスト項目:
  - キャッシュ有効期限: 7日
  - 強制更新オプション: ユーザーが「最新情報を取得」できるか
  - データ鮮度: いつのデータか表示される
```

### 7. パフォーマンステスト

#### TC-23: 応答時間

```
測定対象:
  - 初回カウンセリング: 5～10分
  - ポートフォリオテンプレート: 2～3分
  - 月次フォローアップ: 3～5分

測定方法:
  1. 各ワークフローの総実行時間を計測
  2. ボトルネック（遅い処理）を特定
  3. 許容範囲内か判定

期待結果:
  ✅ 初回: 5～10分以内
  ✅ 続行: 2～5分以内
  ✅ ユーザーが待てる範囲

テスト項目:
  - Web検索: 2～3分 (3～5キーワード並列)
  - WebFetch: 1～2分 (上位5件)
  - Claude API: 1～3分 (内容量による)
  - 総合: 5～10分以内

ボトルネック分析:
  - Web検索タイムアウト: キャッシュで短縮
  - API遅延: リトライ戦略、キューイング
```
