# 副業支援エージェント - ツール統合仕様書

## 📋 概要

このドキュメントでは、副業支援エージェントが使用するツール、API、データソースの詳細な統合方法を定義します。

---

## 🔧 ツール一覧

### 1. Web検索ツール

**用途**: 市場トレンド、相場情報、プラットフォーム分析

#### 1.1 検索キーワード戦略

| 検索カテゴリ | キーワード例 | 目的 |
|-------------|-----------|------|
| **相場・単価** | `{スキル名} 副業 単価 2026` | 報酬水準を把握 |
| | `{スキル名} freelance rate USD JPY` | グローバル相場 |
| **トレンド** | `{業界} 市場動向 2026` | 成長性を確認 |
| | `{スキル名} 需要 求人数 トレンド` | 求人数の推移 |
| **プラットフォーム** | `Upwork {スキル名} top rated` | 成功事例 |
| | `Coconala {スキル名} 実績` | 国内プラットフォーム分析 |
| **リスク** | `{スキル名} 著作権 法的注意` | 法的リスク |
| | `{スキル名} 詐欺 注意` | トラブル事例 |

#### 1.2 検索実行タイミング

```
シーン A: 初回コンサルテーション
  └─ キーワード: 4～6個
     実行タイミング: ユーザー入力後、リアルタイム
     キャッシュ期間: 1週間

シーン B: 定期フォローアップ（毎月）
  └─ キーワード: 2～3個（最新トレンドのみ）
     実行タイミング: 月1回
     キャッシュ期間: 1ヶ月

シーン C: 特定テーマの深掘り
  └─ キーワード: 3～5個
     実行タイミング: ユーザーリクエスト時
     キャッシュ期間: なし（常に最新を取得）
```

#### 1.3 検索結果の処理

```
フロー:
  1. Web検索実行
     └─ 結果: URL、タイトル、スニペット

  2. 自動フィルタリング
     └─ 信頼度スコアで優先順付け
        ├─ 政府・公式機関 (信頼度 95%)
        ├─ 業界レポート・メディア (信頼度 80%)
        ├─ 個人ブログ・SNS (信頼度 50%)
        └─ 不明なソース (信頼度 20%, 削除)

  3. WebFetch で詳細取得（上位5件まで）
     └─ テーブル・数値データを抽出

  4. 情報の統合・要約
     └─ エージェントが分析して提示
```

### 2. WebFetch ツール

**用途**: プラットフォーム・求人情報の詳細データ取得

#### 2.1 対象サイト

| プラットフォーム | URL例 | 抽出データ |
|----------------|------|---------|
| **Upwork** | `https://www.upwork.com/jobs/search/?q={skill}` | 案件数、報酬範囲、スキル要件 |
| **Coconala** | `https://coconala.com/services?keyword={skill}` | サービス数、相場、評価 |
| **クラウドワークス** | `https://crowdworks.jp/public/jobs/search?keywords={skill}` | 案件数、単価、 募集期限 |
| **LANCE** | `https://www.lance.jp/` | 上記同様 |
| **Wantedly** | `https://www.wantedly.com/` | 案件内容、企業評価 |
| **Skillshare** | `https://www.skillshare.com/` | コース需要（教育系向け） |

#### 2.2 抽出フィールド

```javascript
{
  "platform": "Upwork",
  "skills": "Python, Data Analysis",
  "totalJobs": 1250,
  "priceRange": {
    "min": 25,
    "max": 150,
    "currency": "USD",
    "median": 60
  },
  "recentlyPosted": 45,
  "competitionLevel": "medium",
  "trendingKeywords": ["machine learning", "pandas", "visualization"]
}
```

#### 2.3 エラーハンドリング

```
例外シナリオ:
  ├─ 403 Forbidden (ロボット検出)
  │  └─ リトライ待機: 60秒 → 120秒 → キャッシュ使用
  │
  ├─ 404 Not Found
  │  └─ ログに記録、スキップ
  │
  └─ 接続タイムアウト
     └─ 最後の成功キャッシュを使用、警告表示
```

### 3. ローカルファイルツール

**用途**: 既存プロジェクト情報・ナレッジの参照

#### 3.1 参照するファイル

| ファイル | パス | 用途 |
|--------|------|------|
| **AI副業リサーチ** | `products/003_ai_fukugyo/research.md` | 市場情報・単価感 |
| **AI副業マニュアル** | `products/003_ai_fukugyo/manuscript.md` | 教育コンテンツ |
| **成果物カタログ** | `products/MANIFEST.json` | スキル例・プロジェクト参考 |
| **各プロジェクトREADME** | `products/NNN_*/README.md` | 実装例・成功事例 |
| **設定ファイル** | `products/018_side_hustle_agent/config.json` | エージェント設定（後述） |

#### 3.2 データ抽出パターン

```python
# パターン A: 単価情報の抽出
def extract_pricing_from_research():
    """003_ai_fukugyo/research.md から単価情報を抽出"""
    return {
        "月1〜2万円": "初心者が達成可能な目標",
        "月5万円": "専門性の掛け算が必要",
        "主要ジャンル": [...]
    }

# パターン B: スキル例の自動生成
def extract_skill_examples():
    """MANIFEST.json から成功事例を収集"""
    return [
        "001_bg_remover: 画像処理",
        "008_requirements_generator: ドキュメント自動化",
        ...
    ]
```

#### 3.3 キャッシュ戦略

```
更新頻度:
  ├─ research.md: 月1回（時間がある時）
  ├─ MANIFEST.json: 新規プロジェクト時
  ├─ config.json: 毎回（設定変更が頻繁）
  └─ README.md: 参照時に自動再読込

キャッシュ TTL:
  ├─ 静的データ（ジャンル・相場）: 30日
  ├─ 動的データ（トレンド・求人数）: 7日
  └─ 設定: 1時間
```

### 4. Claude API（テキスト生成）

**用途**: ポートフォリオ、営業文、記事執筆

#### 4.1 生成タスク

| タスク | 入力 | 出力 | モデル |
|-------|------|------|-------|
| **プロフィール文生成** | スキル、経歴、強み | 複数パターン（短・中・長） | Claude 3.5 Sonnet |
| **営業メール生成** | 対象クライアント、提案内容 | カスタマイズされたメール | Claude 3.5 Sonnet |
| **SNS投稿案生成** | テーマ、目的、字数 | 複数パターン + ハッシュタグ | Claude 3.5 Haiku |
| **記事執筆** | トピック、構成、字数 | 完成記事 | Claude 3.5 Sonnet |
| **ポートフォリオサイト文** | プロジェクト情報 | SEO対応の説明文 | Claude 3.5 Sonnet |

#### 4.2 プロンプト設計

```
例: プロフィール文生成

システムプロンプト:
  あなたは、フリーランスのプロフィール文作成専門家です。
  ユーザーのスキルと経歴から、複数のプロフィール文を生成してください。
  各パターンは、異なるトーンと対象クライアント層を想定してください。

ユーザープロンプト:
  以下の情報から、Upwork用のプロフィール文を3パターン作成してください。
  
  スキル: Python, Data Analysis
  経歴: 3年のデータアナリスト経験
  実績: 10+件のプロジェクト完了
  強み: 迅速な対応、高品質
  
  パターン:
  1. 「プロ向け」（高単価クライアント向け）
  2. 「親しみやすい」（新規クライアント向け）
  3. 「専門性重視」（企業向け）

出力フォーマット:
  ---
  ### パターン1: プロ向け
  [プロフィール文]
  ---
  ### パターン2: 親しみやすい
  [プロフィール文]
  ---
  ### パターン3: 専門性重視
  [プロフィール文]
```

#### 4.3 使用量管理

```
月別トークン予算: 1,000,000トークン

配分:
  ├─ プロフィール・営業文: 30%
  ├─ SNS投稿: 20%
  ├─ 記事執筆: 30%
  ├─ その他（テンプレート等）: 20%

監視:
  ├─ 週単位で使用量を追跡
  ├─ 80%到達時にアラート
  └─ 100%到達時にリクエスト制限（低優先度タスクは遅延実行）
```

## 🔄 統合ワークフロー

### ワークフロー 1: 初回コンサルテーション

```
ステップ 1: ユーザー入力
  └─ スキル、経歴、目標、制約（テキスト入力）

ステップ 2: ローカルデータ参照
  └─ MANIFEST.json から類似プロジェクトを検索
  └─ 003_ai_fukugyo/research.md から相場情報を抽出

ステップ 3: Web検索実行
  └─ キーワード生成:
     ├─ "{スキル名} 副業 単価 2026"
     ├─ "{業界} 市場動向"
     └─ "{プラットフォーム} {スキル名}"
  └─ 検索実行（3～5キーワード並列実行）
  └─ 上位5件を WebFetch で詳細取得

ステップ 4: 情報統合 & 分析
  └─ Claude が、以下を合成:
     ├─ ローカルデータ（信頼度: 高）
     ├─ Web検索結果（信頼度: 中）
     └─ プラットフォーム分析（信頼度: 中）

ステップ 5: 提案生成
  └─ Claude が「副業機会レポート」を生成:
     ├─ トップ3の機会
     ├─ 実行計画（Week 1-4）
     └─ リスク・コスト分析

ステップ 6: テンプレート生成（オプション）
  └─ ユーザーが「ポートフォリオ案」「営業文」を選択
  └─ Claude が該当テンプレートを生成
```

**所要時間**: 初回は 3～5分（検索 + 分析）

### ワークフロー 2: 進捗フォローアップ

```
ステップ 1: ユーザー入力（毎月）
  └─ 進捗報告:
     ├─ 獲得案件数
     ├─ 実績単価
     ├─ クライアント評価
     └─ 課題・困っていることなど

ステップ 2: 最新トレンド検索（オプション）
  └─ 前回の検索から30日以上経過している場合のみ
  └─ 2～3個の最新キーワードで検索

ステップ 3: 分析 & 改善提案
  └─ Claude が、以下を分析:
     ├─ 目標達成度
     ├─ 単価・案件数の推移
     ├─ 市場の変化
     └─ 改善案

ステップ 4: 次月のアクション計画
  └─ 前月の結果に基づいて、次のステップを提案
```

**所要時間**: 2～3分（キャッシュ活用）

### ワークフロー 3: トピック別ディープダイブ

```
例: 「SNS営業の詳細ガイドが欲しい」

ステップ 1: リクエスト受け取り
  └─ テーマ: SNS営業
  └─ スキル: Python + Data Analysis

ステップ 2: Web検索（必要に応じて）
  └─ "SNS營業 フリーランス 2026"
  └─ "Twitter LinkedIn freelance marketing"

ステップ 3: ガイド生成
  └─ Claude が「SNS営業戦略プラン」を生成:
     ├─ プラットフォーム選定（Twitter vs LinkedIn vs note）
     ├─ 投稿テンプレート（10パターン）
     ├─ ハッシュタグ戦略
     ├─ 運用カレンダー（4週間）
     └─ KPI追跡方法

ステップ 4: 随時サポート
  └─ ユーザーがテンプレートをカスタマイズしながら相談可能
```

## 🛡️ エラーハンドリング & リカバリー

### シナリオ別対応

| 問題 | エラー例 | 対応 |
|------|---------|------|
| **Web検索失敗** | タイムアウト | キャッシュから返す、またはスキップして続行 |
| **WebFetch 403** | ロボット検出 | 待機 → リトライ → キャッシュ使用 |
| **API制限** | レート制限 | キューに入れて後で実行 |
| **ローカルファイル存在しない** | FileNotFound | 警告ログ → スキップ → 代替情報を使用 |
| **Claude API エラー** | 429, 500 | リトライ（指数バックオフ） → ユーザーに通知 |
