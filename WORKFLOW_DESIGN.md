# 副業支援エージェント - ワークフロー設計書

## 📋 概要

このドキュメントでは、副業支援エージェントの主要なワークフロー（業務フロー）を詳細に定義します。

---

## 🔄 メインワークフロー

#### Flow チェックポイント

| ステップ | 検証項目 | 失敗時対応 |
|---------|--------|---------|
| 1 | 入力値の完全性 | エラーメッセージ + 再入力促す |
| 2 | ローカルファイル存在確認 | 警告 + スキップして続行 |
| 3 | Web検索キーワード生成成功 | 結果0件の場合は類似キーワード自動生成 |
| 4 | WebFetch 成功率 | タイムアウト時はキャッシュ利用 |
| 5 | 分析結果の妥当性 | 信頼度スコア 50%未満なら「情報不足」警告 |
| 6 | 提案生成成功 | 失敗時は「カスタマイズ提案」の手動オプション |

### Workflow 2: 段階別サポート - 実行計画フロー

**目的**: ユーザーが実際に行動できるよう、段階的な具体的タスクを提供

**所要時間**: 2～3分（1回目）+ 継続的フォローアップ

```
初期段階 (Week 1-2)
  ├─ [ ] スキルのポートフォリオ作成
  │   ├─ フォーマット選択（GitHub, Behance, 個人ブログ等）
  │   ├─ プロフィール文の作成（テンプレート提供）
  │   ├─ 過去プロジェクト（なければサンプル）の記載
  │   └─ 料金表の作成
  │
  ├─ [ ] プラットフォーム登録
  │   ├─ Upwork (グローバル向け)
  │   ├─ Coconala (国内向け)
  │   └─ クラウドワークス (国内向け)
  │
  └─ [ ] 営業準備
      ├─ 自己紹介文テンプレート確認
      ├─ 案件応募メール雛形作成
      └─ 実績0件対策（サンプルプロジェクト案作成）

成長段階 (Week 3-4)
  ├─ [ ] 営業活動開始
  │   ├─ クラウドソーシング: 毎日 3～5件の案件に応募
  │   ├─ SNS: 週 3回の発信開始
  │   ├─ ネットワーク: コミュニティ参加
  │   └─ リファーラル: 知人への案内
  │
  ├─ [ ] 実績作り
  │   ├─ 単価は低くてもいいから「完了」を重視
  │   ├─ クライアント評価を 5つ星にする
  │   └─ 2～3件の実績を獲得
  │
  └─ [ ] ポートフォリオ更新
      └─ 実績を反映させ、単価を段階的に上げる

安定段階 (Month 2-3以降)
  ├─ [ ] 単価交渉
  │   ├─ 実績がある程度できたら単価を 1.5倍に引き上げ
  │   └─ リピートクライアント優先
  │
  ├─ [ ] 高単価案件へシフト
  │   ├─ 単価 $100/時間以上の案件を狙う
  │   └─ 「専門性」を見える化（ケーススタディ、記事執筆など）
  │
  ├─ [ ] 定期クライアント確保
  │   ├─ 月定額 (retainer) 契約を目指す
  │   └─ 既存クライアントへの提案
  │
  └─ [ ] 事業化検討
      ├─ 継続的か一時的か判断
      ├─ 必要に応じて個人事業主登録
      └─ 青色申告、税務対策

```

## 🔄 エラーハンドリング & フォールバック

### シナリオ別対応

| 問題 | 優先度 | 対応 |
|------|-------|------|
| **Web検索がタイムアウト** | 中 | キャッシュから返す + 警告「最新情報は〜」 |
| **ユーザー入力が不完全** | 高 | エラーメッセージ + 必須項目を赤枠 + 再入力促す |
| **相場情報が見つからない** | 中 | 「情報不足です」+ 代替案提示 + 手動入力フォーム |
| **提案が矛盾している** | 高 | 内部検証後、修正提案または「カスタマイズ相談」に遷移 |
| **Claude API エラー** | 高 | リトライ（最大3回）+ 失敗時は「テンプレート手動入力」 |
