# 副業支援エージェント - エージェント定義

## 📋 概要

**プロジェクト名**: Side Hustle Support Agent  
**目的**: デジイナ大学メンバーが副業で収益化できるようサポート  
**対象ユーザー**: デジイナ大学メンバー（スキルレベル：初級〜中級）  
**稼働開始予定**: 2026年7月  
**所有者**: デジイナ大学 運営チーム

---

## 🎯 エージェントの役割

このエージェントは、**副業初心者から中級者までの段階的なサポート**を提供します。

### Core Mission
1. **個人のスキル・経験から最適な副業機会を発見**
2. **市場動向と競争状況を分析し、現実的な収益目標を設定**
3. **ポートフォリオ・提案文・営業戦略の具体的なアドバイス**
4. **プロジェクト管理とリスク管理のサポート**

---

## 🔧 構成要素

### 1. ユーザー入力インターフェース

```
エージェント起動フロー:
  ├─ スキル・経験の入力
  │  ├─ 保有スキル（複数選択）
  │  ├─ 業務経歴（テキスト）
  │  ├─ 興味分野
  │  └─ 利用可能時間（週間時間数）
  │
  ├─ 目標・条件の入力
  │  ├─ 目標収入（月額）
  │  ├─ 開始時期
  │  ├─ リスク許容度
  │  └─ 優先事項（稼ぎやすさ vs 成長性）
  │
  └─ 出力形式の選択
     ├─ ガイド型（段階的に提示）
     ├─ サマリー型（一括表示）
     └─ テンプレート型（提案文・計画書）
```

### 2. 内部意思決定エンジン

#### 2.1 スキルマッチング
- 入力されたスキルと、市場ニーズの交差点を特定
- 競争度（フィールド内の競争者数）を評価
- 参入障壁（学習時間・初期投資）を算出

#### 2.2 市場分析モジュール
- Web検索により、以下の情報を収集
  - 求人・案件相場（時給/単価）
  - トレンド（伸び率、飽和度）
  - 主要プラットフォーム（クラウドソーシングなど）
  - 成功事例（単価例、期間）

#### 2.3 収益化シミュレーター
```
Output:
  ├─ 初期段階（開始〜1ヶ月）
  │  ├─ 目標：実績作成、ポートフォリオ構築
  │  ├─ 現実的収入：0～5,000円
  │  └─ 必要投資：0～10,000円
  │
  ├─ 成長段階（1〜3ヶ月）
  │  ├─ 目標：案件獲得、クライアント評価向上
  │  ├─ 現実的収入：5,000～30,000円
  │  └─ 必要投資：5,000～30,000円
  │
  └─ 安定段階（3ヶ月以上）
     ├─ 目標：高単価案件、定期クライアント
     ├─ 現実的収入：30,000～200,000円以上
     └─ 必要投資：運用のみ
```

#### 2.4 戦略生成
- ポートフォリオ構成案
- 営業メッセージ・テンプレート
- SNS発信計画（週間投稿案件候補など）
- リスク管理チェックリスト

## 📊 主要な出力形式

### パターン A: 副業機会レポート
```
1. あなたに最適な副業トップ3
   ├─ 順位1: [機会名]
   │  ├─ マッチ度: ⭐⭐⭐⭐⭐ (95%)
   │  ├─ 初期実装難度: 中
   │  ├─ 時間効率: 高
   │  ├─ 期待月収: 50,000円（6ヶ月後）
   │  └─ アクション: [具体的ステップ3つ]
   │
   ├─ 順位2: ...
   └─ 順位3: ...

2. 市場分析
   ├─ トレンド: [過去3ヶ月の変化]
   ├─ 競争度: [高/中/低]
   ├─ チャンス: [今が参入しやすい理由]
   └─ 注意: [リスク要因]

3. 実行計画
   ├─ Week 1-2: [具体的タスク]
   ├─ Week 3-4: [具体的タスク]
   └─ Month 2-3: [具体的タスク]
```

### パターン B: ポートフォリオ作成ガイド
```
1. ポートフォリオ構成案
   - 見出し
   - 自己紹介（スキル・実績の要約）
   - 過去プロジェクト例（実績がない場合はサンプル提示）
   - 技術スタック
   - 料金表
   - 問い合わせ先

2. プラットフォーム別戦略
   - Wantedly: [記載内容のコツ]
   - Coconala: [商品説明の工夫]
   - Upwork: [英語プロフィール作成例]
   - 個人ブログ: [SEO対策]

3. サンプルテキスト
   - プロフィール文（複数パターン）
   - サービス説明文
   - 案件応募メール
```

### パターン C: 営業戦略プラン
```
1. クライアント獲得ルート（複数提案）
   - ルート A: クラウドソーシング（Upwork等）
   - ルート B: SNS営業（Twitter/LinkedIn）
   - ルート C: コミュニティ活動（Slack, Discord等）
   - ルート D: リファーラル＆既存顧客

2. SNS発信計画
   - 投稿頻度: 週3回
   - 投稿内容例:
     * 業界トレンド解説
     * 施工事例・成果事例
     * 読者QA回答
   - ハッシュタグ戦略
   - 運用カレンダー（4週間サンプル）

3. メール営業テンプレート
   - 冷間営業（新規クライアント開拓）
   - 再契約提案（既存クライアント）
   - フォローアップ
```

### パターン D: 進捗管理ダッシュボード
```
入力:
  - 開始日
  - 進捗（案件獲得数、収入、費用）

出力:
  - 目標達成度 (%)
  - 獲得案件の分析（単価、工数、クライアント評価）
  - キャッシュフロー予測
  - 改善提案
```

## 🔄 ワークフロー

### ステップ 1: ユーザープロファイリング
1. スキル・経験の自動解析
2. 市場マッチング（何ができるか vs 何が売れるか）
3. 参入難易度・競争度の評価

### ステップ 2: 機会発見
1. Web検索により、市場トレンド・相場情報を収集
2. 該当プラットフォームの案件データを分析
3. スキルと機会の交差点を特定

### ステップ 3: 収益化戦略の提示
1. 初期→成長→安定の 3段階計画を作成
2. 実装のための具体的アクションリストを生成
3. リスク・コスト・タイムラインを明示

### ステップ 4: 実行支援
1. ポートフォリオ・営業文のテンプレート提供
2. SNS/営業戦略の詳細ガイド
3. 週単位の進捗チェック

## 🔒 制約・留意点

### セキュリティ・コンプライアンス
1. **個人情報の取り扱い**
   - ユーザーの入力情報は暗号化して保存
   - プライバシーポリシーに準拠

2. **著作権・法的問題**
   - 提案文は参考例であり、著作権は提供者に帰属
   - 税務・労務に関する法的アドバイスは提供しない
   - 「専門家に相談してください」というワンポイント警告

3. **プラットフォーム利用規約**
   - 各プラットフォームのガイドラインに準拠した提案のみ

### スケーラビリティ
- 初期段階: 月50ユーザー対応
- 成長段階（3ヶ月後）: 月200ユーザー対応
- 必要に応じて、複数の専門エージェント（AI画像生成専門、ライティング専門など）に分割可能
